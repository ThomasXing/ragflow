# RAGFlow 镜像构建流程梳理

## 问题分析

当前存在多个构建脚本和Dockerfile，导致混乱：

### 1. 构建脚本冲突（4个文件）

| 文件 | 用途 | 问题 |
|------|------|------|
| `build.sh` | Go服务构建（非Docker） | 功能明确，无冲突 |
| `build-ragflow-linux-cpu.sh` | 通用Linux CPU镜像构建 | 最新版本，功能最全 |
| `build-ragflow-cpu.sh` | 简单的CPU镜像构建 | 简化版本，功能不完整 |
| `build-linux-cpu.sh` | ARM64特殊处理构建 | 临时修改Dockerfile，风险高 |

### 2. Dockerfile混乱（6个文件）

| 文件 | 用途 | 状态 |
|------|------|------|
| `Dockerfile` | 标准Dockerfile | 依赖`infiniflow/ragflow_deps:latest` |
| `Dockerfile.cpu-universal` | 通用CPU Dockerfile | 无外部依赖，最新 |
| `Dockerfile.arm64.final` | ARM64最终版本 | 从Dockerfile.arm64.fixed优化 |
| `Dockerfile.arm64.fixed` | ARM64修复版本 | 存在语法问题 |
| `Dockerfile.arm64` | ARM64原始版本 | 可能有问题 |
| `Dockerfile.deps` | 依赖镜像构建 | 独立用途 |

### 3. 核心问题

1. **依赖镜像不可用**：标准`Dockerfile`依赖`infiniflow/ragflow_deps:latest`，在离线或无网络环境失败
2. **脚本功能重叠**：多个脚本做类似事情，参数不一致
3. **ARM64支持混乱**：有多个ARM64 Dockerfile，不清楚哪个是正确的

## 解决方案

### 方案A：保持向后兼容（推荐）

1. **统一入口脚本**：`build-docker.sh` - 智能选择构建方式
2. **分层构建结构**：
   - `Dockerfile.deps` → 基础依赖镜像
   - `Dockerfile` → 生产镜像（依赖deps镜像）
   - `Dockerfile.cpu-universal` → CPU专用镜像（无外部依赖）
3. **明确使用场景**：
   - **开发环境**：使用`Dockerfile.cpu-universal`
   - **生产环境**：构建`deps`镜像 + `Dockerfile`
   - **离线环境**：使用`Dockerfile.cpu-universal`

### 方案B：完全重构

1. **单一日志**：`build-docker.sh`
2. **单一日志**：`Dockerfile`（支持多架构）
3. **统一参数**：通过ARG控制构建方式

## 推荐使用流程

### 快速构建（CPU/开发环境）

```bash
# 使用通用CPU镜像（推荐）
./build-ragflow-linux-cpu.sh [amd64|arm64] [镜像标签] [是否推送] [仓库地址]

# 示例
./build-ragflow-linux-cpu.sh                # 自动检测架构
./build-ragflow-linux-cpu.sh amd64          # 构建x86_64
./build-ragflow-linux-cpu.sh arm64 mytag    # 构建ARM64带标签
```

### 标准构建（生产环境）

```bash
# 1. 构建依赖镜像
docker build -f Dockerfile.deps -t infiniflow/ragflow_deps .

# 2. 构建生产镜像
docker build -f Dockerfile -t infiniflow/ragflow:nightly .
```

### 特殊需求

- **ARM64 Mac (M1/M2)**：使用`Dockerfile.cpu-universal`或`build-ragflow-linux-cpu.sh arm64`
- **离线环境**：使用`Dockerfile.cpu-universal`
- **多架构**：使用`build-ragflow-linux-cpu.sh`支持多架构manifest

## 各Dockerfile对比

| 特性 | Dockerfile | Dockerfile.cpu-universal | Dockerfile.arm64.final |
|------|------------|--------------------------|------------------------|
| 外部依赖 | 有（infiniflow/ragflow_deps） | 无（直接下载） | 无（基于cpu-universal优化） |
| 构建速度 | 慢（需先构建deps） | 快（直接下载） | 快 |
| 镜像大小 | 小（多阶段） | 大（单阶段） | 大 |
| 生产就绪 | ✅ 是 | ⚠️ 开发环境 | ⚠️ 开发环境 |
| ARM64支持 | ❌ 需要修改 | ✅ 支持 | ✅ 专门优化 |
| 离线构建 | ❌ 不支持 | ✅ 支持（需要缓存） | ✅ 支持 |
